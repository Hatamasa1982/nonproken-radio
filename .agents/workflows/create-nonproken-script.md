# ノンプロ研ラジオ準備ワークフロー（/create-nonproken-script）

次回の「ノンプロ研ラジオ」の収録と編集に向けた準備を行います。AIアシスタントは、以下のステップを順番に実行し、必要項目を準備してください。

1. 最初に、ユーザーに以下の情報だけを質問してください。
  - **「第何回の準備をはじめますか？」**

2. ユーザーから「放送回数（例：14）」の回答をもらったら、AIは過去の法則（第14回が2026/04/13で、毎週月曜日に放送される）から**「放送日（YYYYMMDD形式）」を自動で計算**してください。

3. 放送日を計算できたら、**「〇月〇日（〇）の放送ですね！ 取り上げるnoteのURLを教えてください」**とユーザーに質問してください。

4. ユーザーからURL情報を受け取ったら、`read_url_content` ツールを使って指定されたnote記事の内容を読み取ってください。記事の内容から、以下の5つの情報をAIとして取得（または判断・要約）してください。
  - note記事のタイトル
  - 放送日
  - シリーズ名（例：隣のAIエージェント）
  - シリーズ内の回数（例：12）
  - 放送の概要（どんな話をしているかの要約）
  - ※ただし「深掘りポイント」はゲストが実際に手動で考えるため、ここでの抽出や作成は不要です。

5. 今回の「放送回」が奇数か偶数かを判定してください。
  - **奇数回**の場合：ホストが「はた」、ゲストが「空回り父さん」。原本は `./台本原本/奇数回（ホスト：はた）.md` 
  - **偶数回**の場合：ホストが「空回り父さん」、ゲストが「はた」。原本は `./台本原本/偶数回（ホスト：空回り父さん）.md`

6. `run_command` ツールを使って、以下のパスに新しいフォルダを作成してください（すでに存在する場合はスキップ）。
  - パス：`./#XXX（放送日YYYYMMDD）`
  - ※`#XXX` は放送回を3桁ゼロ埋めした数字（例：#014）、YYYYMMDDはステップ2で計算した日付です。

7. 続けて `run_command` ツールを使い、作成した「#XXX（放送日YYYYMMDD）」フォルダ内に、ステップ5で選択した「台本原本（.md）」を「#XXX台本.md」というファイル名でコピーしてください。

8. コピーした新しい台本ファイルの中身を、以下の内容で書き換えてください。（`replace_file_content` などを使い適宜更新）
  - 「0. 概要」の「放送回」と「放送日」：ステップ1で取得した回数と、ステップ2で計算した放送日
  - 「0. 概要」の「取り上げたnote：」のあとに、ステップ3でユーザーから提供されたnote記事のURLを追加
  - 「3. 本編」の挨拶以降の部分：
    - 「・放送日：」のあとに実際の日付を追加
    - 「・シリーズ名（シリーズ回数）：」のあとに、ステップ4で取得したシリーズ名と回数を追加
    - 「・タイトル：」のあとに、ステップ4で取得したnote記事のタイトルを追加
    - 「放送の概要について教えてください。」のあとの回答（回答）部分に、ステップ4で作成した「放送の概要」を書き込み
    - 「深堀りポイント」のリスト（・）は、そのまま残しておいてください（ゲストが手書きするため）。

9. すべての準備が完了したら、作業した内容とフォルダの保存場所を、ユーザーにやさしい日本語で報告してください。
